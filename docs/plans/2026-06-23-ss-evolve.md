# ss-evolve Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `/ss-evolve` — a deterministic ledger-pattern detector (`scripts/ss-evolve` + `.ps1`) with a risk-tiered auto-apply, plus a skill that adds LLM-authored drafting — so the project improves its own setup from accumulated loop signal.

**Architecture:** `scripts/ss-evolve` scans `.superstack/ledger.jsonl` for two countable patterns (`skipped-phase`, `failing-gate`) above a threshold, emitting findings (`--json`/human). `--apply` appends a templated insight to `CONTEXT.md` and makes one revertable `chore(evolve):` commit per new finding, deduped via `.superstack/evolve-state`. The `/ss-evolve` skill runs detection and authors richer changes, routing brand-new skills to `.superstack/proposals/` (never auto-committed).

**Tech Stack:** POSIX bash, PowerShell, `jq`, git. No new dependencies.

## Global Constraints

- Cross-platform: `scripts/ss-evolve` (bash) and `scripts/ss-evolve.ps1` produce identical findings + identical `--apply` output for the same ledger/repo.
- **ASCII-only output** (no em-dash/middot/Unicode-minus) — byte-parity across twins, same rule as `ss-report`.
- Scripts begin with the shebang then `# SPDX-License-Identifier: MIT`. Read the ledger via stdin redirection (`jq … < "$ledger"`).
- Ledger dir overridable via `SUPERSTACK_DIR` (default `.superstack`). Threshold from `.superstack/config` key `evolve_threshold` (default `3`).
- Finding id = `<type>:<phase>` (`skipped:secure`, `failing:review`). Dedup via `.superstack/evolve-state` (newline-delimited ids).
- `--apply` touches ONLY `CONTEXT.md` (+ state + a ledger `note`); one `chore(evolve):` commit per new finding. Never creates skills.
- `ss-evolve` never gates: exit 0 normally; exit 1 on unknown flag; exit 2 if `jq` missing.
- Skill names start `ss-`; description starts `Use `, 40–500 chars, one H1. Conventional commits; bash exec bit via `git update-index --chmod=+x`.
- `bash scripts/lint-skills.sh` (→ `OK: 22 skill(s)`) and `bash tests/run.sh` (→ `ALL TESTS PASS`) stay green.

```bash
#!/usr/bin/env bash
# SPDX-License-Identifier: MIT
# Detect strong patterns in the loop ledger; optionally auto-apply low-risk fixes.
# Usage: ss-evolve [--json] [--new-only] [--apply] [--dry-run]
set -uo pipefail
dir="${SUPERSTACK_DIR:-.superstack}"
ledger="$dir/ledger.jsonl"; config="$dir/config"; state="$dir/evolve-state"
json=0; newonly=0; apply=0; dry=0
for a in "$@"; do
  case "$a" in
    --json) json=1;;
    --new-only) newonly=1;;
    --apply) apply=1;;
    --dry-run) dry=1;;
    *) echo "ss-evolve: unknown flag '$a' (usage: ss-evolve [--json] [--new-only] [--apply] [--dry-run])" >&2; exit 1;;
  esac
done
command -v jq >/dev/null 2>&1 || { echo "ss-evolve: jq required" >&2; exit 2; }

threshold=3
[ -f "$config" ] && { v="$(grep -E '^evolve_threshold=' "$config" | tail -1 | cut -d= -f2-)"; [ -n "$v" ] && threshold="$v"; }

findings=""
if [ -f "$ledger" ]; then
  findings="$(jq -rn --argjson th "$threshold" '
    [inputs] as $all
    | ( $all | map(select(.event=="skip")) | group_by(.phase) | map(
          { type:"skipped", phase:.[0].phase, count:length,
            reason:( [.[].note] | map(select(. != null and . != "")) | if length==0 then "" else (group_by(.)|max_by(length)|.[0]) end ) } ) ) as $s
    | ( $all | map(select(.event=="gate" and .status=="fail")) | group_by(.phase) | map(
          { type:"failing", phase:.[0].phase, count:length, reason:"" } ) ) as $f
    | ($s + $f) | .[] | select(.count >= $th)
    | [.type,.phase,(.count|tostring),.reason] | @tsv
  ' < "$ledger" | tr -d '\r')"
fi

seen() { [ -f "$state" ] && grep -qxF "$1" "$state"; }

apply_one() { # type phase count reason
  local id="$1:$2" line
  case "$1" in
    skipped) line="- **\`$2\` is routinely skipped** (${3}x${4:+; usual reason: \"$4\"}). If that is expected here, keep recording the skip reason or drop \`$2\` from \`.superstack/config\` \`mandatory_phases\`; otherwise it is a process gap to close.";;
    failing) line="- **\`$2\` gate often fails first pass** (${3}x). Recurring friction - see the ledger notes; consider tightening the upstream phase or adding a checklist.";;
  esac
  if [ "$dry" -eq 1 ]; then echo "[dry-run] $id -> append to CONTEXT.md + commit 'chore(evolve): document $id'"; return; fi
  [ -f CONTEXT.md ] || printf '# Context\n' > CONTEXT.md
  grep -qxF '## Evolved insights' CONTEXT.md || printf '\n## Evolved insights\n' >> CONTEXT.md
  printf '%s\n' "$line" >> CONTEXT.md
  mkdir -p "$dir"; printf '%s\n' "$id" >> "$state"
  local sib; sib="$(cd "$(dirname "$0")" && pwd)/ledger"
  [ -f "$sib" ] && bash "$sib" evolve note na "applied $id" >/dev/null 2>&1
  git add CONTEXT.md >/dev/null 2>&1 && git commit -q -m "chore(evolve): document $id" >/dev/null 2>&1
  echo "applied $id (revert with: git revert HEAD)"
}

items=""; n=0
while IFS=$'\t' read -r t p c r; do
  [ -n "$t" ] || continue
  id="$t:$p"
  { [ "$newonly" -eq 1 ] || [ "$apply" -eq 1 ]; } && seen "$id" && continue
  n=$((n + 1))
  if [ "$apply" -eq 1 ]; then
    apply_one "$t" "$p" "$c" "$r"
  elif [ "$json" -eq 1 ]; then
    items="${items:+$items,}$(jq -cn --arg id "$id" --arg t "$t" --arg p "$p" --argjson c "$c" --arg r "$r" '{id:$id,type:$t,phase:$p,count:$c,reason:$r}')"
  else
    printf -- '- [%s] %s (x%s)%s\n' "$t" "$p" "$c" "${r:+ - reason: \"$r\"}"
  fi
done <<EOF
$findings
EOF

if [ "$apply" -eq 0 ] && [ "$json" -eq 1 ]; then printf '[%s]\n' "$items"
elif [ "$apply" -eq 0 ] && [ "$n" -eq 0 ]; then echo "ss-evolve: no patterns at or above threshold $threshold"
elif [ "$apply" -eq 1 ] && [ "$n" -eq 0 ]; then echo "ss-evolve: nothing new to apply"
fi
exit 0
```

Create `scripts/ss-evolve.ps1`:

```powershell
#!/usr/bin/env pwsh
# SPDX-License-Identifier: MIT
# Detect ledger patterns; optionally auto-apply low-risk fixes. Usage: ss-evolve.ps1 [-Json] [-NewOnly] [-Apply] [-DryRun]
param([switch]$Json, [switch]$NewOnly, [switch]$Apply, [switch]$DryRun)
$ErrorActionPreference = 'Stop'
$dir = if ($env:SUPERSTACK_DIR) { $env:SUPERSTACK_DIR } else { '.superstack' }
if ($dir -match '^/[a-zA-Z]/') { try { $dir = (& cygpath -w $dir 2>$null).Trim() } catch {} }
$ledger = Join-Path $dir 'ledger.jsonl'; $config = Join-Path $dir 'config'; $state = Join-Path $dir 'evolve-state'

$threshold = 3
if (Test-Path $config) {
  $m = Select-String -Path $config -Pattern '^evolve_threshold=(.*)$' | Select-Object -Last 1
  if ($m) { $threshold = [int]$m.Matches.Groups[1].Value }
}

$findings = @()
if (Test-Path $ledger) {
  $all = @(Get-Content $ledger | Where-Object { $_ } | ForEach-Object { $_ | ConvertFrom-Json })
  $skips = @($all | Where-Object { $_.event -eq 'skip' } | Group-Object phase | Sort-Object Name | ForEach-Object {
    $notes = @($_.Group | ForEach-Object { $_.note } | Where-Object { $_ })
    $reason = if ($notes.Count) { ($notes | Group-Object | Sort-Object Count, Name -Descending | Select-Object -First 1).Name } else { '' }
    [pscustomobject]@{ id = "skipped:$($_.Name)"; type = 'skipped'; phase = $_.Name; count = $_.Count; reason = $reason } })
  $fails = @($all | Where-Object { $_.event -eq 'gate' -and $_.status -eq 'fail' } | Group-Object phase | Sort-Object Name | ForEach-Object {
    [pscustomobject]@{ id = "failing:$($_.Name)"; type = 'failing'; phase = $_.Name; count = $_.Count; reason = '' } })
  $findings = @($skips) + @($fails) | Where-Object { $_.count -ge $threshold }
}

function Seen([string]$id) { (Test-Path $state) -and (Select-String -Path $state -Pattern ([regex]::Escape($id)) -SimpleMatch -Quiet) }

$active = @($findings | Where-Object { -not (($NewOnly -or $Apply) -and (Seen $_.id)) })

if ($Apply) {
  $applied = 0
  foreach ($f in $active) {
    $line = if ($f.type -eq 'skipped') {
      "- **``$($f.phase)`` is routinely skipped** ($($f.count)x" + $(if ($f.reason) { "; usual reason: ""$($f.reason)""" } else { '' }) + "). If that is expected here, keep recording the skip reason or drop ``$($f.phase)`` from ``.superstack/config`` ``mandatory_phases``; otherwise it is a process gap to close."
    } else {
      "- **``$($f.phase)`` gate often fails first pass** ($($f.count)x). Recurring friction - see the ledger notes; consider tightening the upstream phase or adding a checklist."
    }
    if ($DryRun) { Write-Output "[dry-run] $($f.id) -> append to CONTEXT.md + commit 'chore(evolve): document $($f.id)'"; continue }
    if (-not (Test-Path CONTEXT.md)) { Set-Content CONTEXT.md "# Context" -Encoding utf8 }
    if (-not (Select-String -Path CONTEXT.md -Pattern '^## Evolved insights$' -Quiet)) { Add-Content CONTEXT.md "`n## Evolved insights" }
    Add-Content CONTEXT.md $line
    New-Item -ItemType Directory -Force -Path $dir | Out-Null; Add-Content $state $f.id
    $sib = Join-Path $PSScriptRoot 'ledger'; if (Test-Path $sib) { try { bash $sib evolve note na "applied $($f.id)" *>$null } catch {} }
    git add CONTEXT.md *>$null; git commit -q -m "chore(evolve): document $($f.id)" *>$null
    Write-Output "applied $($f.id) (revert with: git revert HEAD)"; $applied++
  }
  if (-not $DryRun -and $applied -eq 0) { Write-Output "ss-evolve: nothing new to apply" }
}
elseif ($Json) {
  Write-Output (@($active | ForEach-Object { [pscustomobject]@{ id = $_.id; type = $_.type; phase = $_.phase; count = $_.count; reason = $_.reason } }) | ConvertTo-Json -Compress -AsArray)
}
else {
  if ($active.Count -eq 0) { Write-Output "ss-evolve: no patterns at or above threshold $threshold" }
  foreach ($f in $active) {
    $sfx = if ($f.reason) { " - reason: ""$($f.reason)""" } else { '' }
    Write-Output "- [$($f.type)] $($f.phase) (x$($f.count))$sfx"
  }
}
```
